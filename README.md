# バーコードダンジョン

身近な商品のバーコードと現在地（GPS）を組み合わせるローグライク風 Web アプリ（Vite + Vanilla JS）。

- **歩く** → 地図上の決まった場所にダンジョンが固定湧き（ポケGO風）
- **スキャン** → バーコード種別に応じたアイテムを獲得（装備・回復薬・巻物）
- **戦う** → ダンジョン内のモンスターと自動戦闘風コマンドバトル

## 開発・実行

```bash
npm install
npm run dev      # https://localhost:5173/  （HTTPS自己署名・LAN公開）
npm run build
npm run preview  # ビルド成果物の確認 https://localhost:4173/
```

スマホ実機デバッグの際は同一LANから `https://<PCのIP>:5173/` でアクセス（証明書警告は許可）。カメラ・位置情報はHTTPS必須。

## デバッグモード

URL に `?debug=1` を付けて起動すると、画面右上にデバッグパネルが出ます。

```
https://localhost:5173/?debug=1
```

本番用ブックマーク（`/`）とデバッグ用ブックマーク（`/?debug=1`）を分けて使うことを想定。

| 機能 | 説明 |
|------|------|
| **モックスキャン** | テキスト欄にバーコード文字列、セレクトでフォーマットを選んでクリック → カメラを使わずに同じ獲得フローが流れる |
| **モックGPS** | 緯度経度を直接入力 → プレイヤー位置を強制移動。実GPSが使えない PC でも各地のダンジョン・距離挙動を再現できる |
| **入場距離(80m)を無視** | 遠くのダンジョンにも入れる |

> 状態は URL クエリでのみ判定するため、デバッグ操作の影響は本番ブックマークには波及しません。<br>
> ※ ただし現状はゲームデータをローカルに永続化していないため、リロードでクリア状態は毎回リセットされます。

## ゲームの流れ

1. マップ画面に GPS で現在地が表示される
2. 周辺のダンジョンが自動でピン表示（**約200m格子に60%確率で出現**）
3. ピンをタップ → **80m以内にいれば「入場する」ボタン** が押せる（外側だと「距離XXm」表示）
4. ダンジョン突入：HPは全回復、入場前のインベントリと装備をスナップショット
5. 各フロアを攻略、最終フロアでボス撃破 → 攻略成功
6. 敗北時はダンジョン中に拾った装備とアイテムは失われる（入場時の状態に戻る）
7. **攻略済ダンジョンは再戦可能**（範囲内なら何度でも）
8. マップに戻ってバーコードをスキャンすると **アイテム獲得**

## 対応バーコード形式と挙動

`@zxing/browser` を `EAN_13 / EAN_8 / UPC_A / UPC_E / CODE_128 / ITF` の数字系に絞って起動。アプリ側でさらに **「数字8〜20桁」** に限定（QRコード・英数Code39等は無視）。

| 形式 | 桁数 | 想定用途 | カテゴリ | アプリでの扱い |
|-----|------|---------|---------|------------|
| **JAN-13 / EAN-13** | 13桁 | 日本・世界の商品バーコード | `product` | 通常レア度のアイテム |
| **JAN-8 / EAN-8** | 8桁 | 小型商品の短縮版 | `product` | 通常レア度のアイテム |
| **UPC-A** | 12桁 | 北米の商品 | `product` | 通常レア度のアイテム |
| **UPC-E** | 6〜8桁 | UPC-A の短縮版 | `product` | 通常レア度のアイテム |
| **Code 128（数字）** | 可変 | レシート・配送伝票・チケット | `receipt` | **レア度+1段階の底上げ** |
| **ITF** | 偶数桁 | 書籍流通・段ボール | `receipt` | **レア度+1段階の底上げ** |

> JANコードは EAN-13 と完全互換（日本コードは先頭が `45` または `49`）。

## バーコード → アイテム生成ルール

スキャンで得られるアイテムは **完全に決定論的**（同じバーコードなら同じアイテム）。

### バーコード桁の役割（13桁正規化後）

8〜12桁のコードは内部で **先頭ゼロ埋めで13桁化** されてから読まれます。

| 桁位置 | 影響先 | ロジック |
|-------|--------|---------|
| **4〜5桁目** | アイテム属性 | `% 6` で属性選択 |
| **全桁合計** | アイテム種別 | `% 4` で 武器/防具/回復薬/巻物 |
| **末尾1桁** | レアリティ | 表参照（レシート系は+1段階） |

### レアリティ（末尾1桁）

| 末尾桁 | レアリティ | ステータス倍率 | スキル付与 |
|-------|----------|------------|---------|
| 0〜4 | コモン (灰) | ×1.0 | なし |
| 5〜7 | レア (青) | ×1.7 | 1種類（クリ率/反射 等） |
| 8 | エピック (紫) | ×2.6 | 1種類（高威力スキル） |
| 9 | レジェンド (金) | ×4.2 | 1種類（強力固有スキル） |

> Code 128 / ITF（レシート系）は上記の **末尾桁から+1段階上**のレア度に底上げ。

### アイテム種別（全桁合計 % 4）

| 値 | 種別 | 内容 |
|---|------|------|
| 0 | 武器 | 短剣/剣/斧/槍/魔法杖。属性つき・レア度で名前接尾辞・付属スキルが変化 |
| 1 | 防具 | 盾/鎧/マント。属性つき・レア度で名前接尾辞・付属スキルが変化 |
| 2 | 回復薬 | 小/中/大。HP回復量がレアリティ倍率で増加 |
| 3 | 巻物 | 炎/氷/雷/闇/光。敵への属性ダメージ |

### 装備の付属スキル例

```
コモン:    火の剣              ATK +5 （スキルなし）
レア:      火の剣・改          ATK +8 / 鋭刃: クリティカル率+15%
エピック:  火の剣・烈          ATK +12 / 烈火連斬: 攻撃時+7火ダメージ
レジェンド:火の剣・神威        ATK +20 / 龍王の咆哮: 攻撃時+15火ダメージ
```

> 名前の接尾辞・付属スキル名はレア度別の候補から決定論的に選択されます（同じバーコードなら毎回同じ）。

## ダンジョン生成（位置ベース固定湧き）

地図は **約200m × 200mのグリッド** に区切られ、各セルは `hash("grid:<gx>:<gy>")` をシードにして決定論的にダンジョンが配置されます。

| 要素 | ロジック |
|------|---------|
| 出現可否 | 各セル60%でダンジョン出現 |
| 配置位置 | セル内のランダム位置（中央寄り） |
| テーマ | 5種類（古代遺跡/氷の洞窟/溶岩洞/魔法図書館/毒の沼地） |
| フロア数 | 3〜5 |
| 難易度 | ⭐〜⭐⭐⭐ |
| モンスター族 | 10種類からランダム |
| 属性 | 6属性からランダム |
| レアリティ基準 | コモン45% / レア35% / エピック15% / レジェンド5% |
| 入場可能距離 | プレイヤー位置から **80m以内** |

### モンスター族（10種）

| 族 | 絵文字 |
|----|------|
| スライム | 🟢 |
| ゴブリン | 👺 |
| コウモリ | 🦇 |
| スケルトン | 💀 |
| フェニックス | 🔥 |
| アイスウルフ | 🐺 |
| サンゴリラ | 🦍 |
| ウィスプ | ✨ |
| ゾンビ | 🧟 |
| ドラゴン | 🐉 |

### 属性とスキル

| 属性 | スキル | 効果 |
|------|------|------|
| 火 | 炎の息 | ATK ×2.0 |
| 水 | 水流 | ATK ×1.5 |
| 地 | 岩石投げ | ATK ×2.5 |
| 風 | 竜巻 | ATK ×1.8 |
| 光 | 聖光 | 自HP +25% |
| 闇 | 呪い | ATK ×1.2 + 毒 |

ボスは基準レアリティから **+1段階** 昇格（コモン→レア、…）。

## 開発用メモ

### 新機能を追加するときは「デバッグモード対応」を必ず考慮する

このアプリは GPS とカメラに依存するため、**デバッグモード（`?debug=1`）から動作確認できない機能は実機テスト無しで検証不能** になります。新機能を追加する際は、以下を必ず一緒に実装してください：

| 機能の種類 | デバッグ側で必要な対応 |
|-----------|---------------------|
| **GPSに関わるもの**（移動・距離・スポーン等） | モックGPSの座標で動作することを確認。必要なら `setPlayerPosition` を介して位置を動かせるように |
| **カメラ・スキャンに関わるもの**（読取・認識・入力経路） | モックスキャンの入力（テキスト＋フォーマット）で同じフローが流れるように。実カメラに依存する処理を必ず分離 |
| **永続化・ストレージに関わるもの** | デバッグ用ストレージキーは本番と分離（例: `debug:` プレフィックス）。リセットボタンをデバッグパネルに追加 |
| **時間・期間に関わるもの**（クールダウン等） | 時刻を倍速にするか、即時化するトグルをデバッグパネルへ |
| **戦闘・モンスター関連** | 戦闘テスト用の即時遭遇ボタン、HPやATK上書き、敵生成パラメータ強制指定など |
| **画面遷移・モーダル** | 各画面に直接ジャンプできるショートカットをデバッグパネルへ |

**実装パターン**:

1. 機能のコアロジックを純粋関数化（環境依存しない）
2. 環境依存部分（GPS/カメラ/storage）は最小に閉じ込める
3. デバッグパネルから上記の差し替え点を直接叩けるようにする
4. README のデバッグモード節に新規操作を追記

> 「実機にスマホを持って外を歩かないと検証できない」状況を作らないこと。**PCのデスクトップで一周遊びながらデバッグできる状態を維持** が原則です。

## ファイル構成

```
src/
  main.js       画面遷移・全体制御・スキャン→アイテム獲得フロー
  scanner.js    カメラ + ZXing（フォーマット情報を返す）
  map.js        Leaflet地図 + GPS固定湧きピン管理
  generator.js  グリッドダンジョン生成 + モンスター生成 + 距離計算
  items.js      バーコード → アイテム（武器/防具/薬/巻物）+ レア度別スキル
  dungeon.js    ダンジョンマップ生成・描画
  battle.js     バトルロジック・UI更新
  rng.js        Mulberry32 シード乱数 + ハッシュ
```
