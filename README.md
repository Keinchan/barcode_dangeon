# バーコードダンジョン

身近な商品のバーコードと現在地（GPS）を組み合わせるローグライク／戦闘 RPG Web アプリ（Vite + Vanilla JS + Firebase）。

- **歩く** → 地図上の約 200m 格子にダンジョンが固定湧き（ポケ GO 風）
- **遭遇する** → 80m 圏内に道端モンスター・強敵・宝箱・行商人がポップ
- **スキャン** → バーコードから武器／防具／薬／巻物／鍵／技の書／伝説の書／バーコードダンジョン入口 等を獲得
- **潜る** → 21×19 マスのターン制ローグライク。十字キー＋技スロット 4 個で立ち回る
- **対戦／協力する** → Firebase Firestore 経由でオンライン 1v1 PvP もしくは 2 人協力でボスを討伐

本番 URL: https://barcode-d1c01.web.app

## 開発・実行

```bash
npm install
npm run dev      # https://localhost:5173/  （HTTPS 自己署名・LAN 公開）
npm run build    # dist/ を生成
npm run preview  # ビルド成果物の確認 https://localhost:4173/
```

スマホ実機デバッグの際は同一 LAN から `https://<PCのIP>:5173/` でアクセス（自己署名証明書の警告は許可）。カメラ・位置情報は HTTPS 必須。

### デプロイ（Firebase Hosting）

```bash
npm run build
firebase deploy --only hosting           # dist/ をホスティングへ
firebase deploy --only firestore:rules   # firestore.rules を反映
```

`.firebaserc` / `firebase.json` 同梱。プロジェクト: `barcode-d1c01`。

## ゲームの流れ

1. **タイトル** で Email/パスワード または Google アカウントでログイン（Firebase Auth）。セーブはクラウドに保存され、別端末でも継続。
2. **マップ画面** に GPS で現在地が表示され、周辺のダンジョンと道端エンカウントがピン表示される。
3. **ピンをタップ** → 80m 以内なら入場可。距離 ON で入場ボタン、OFF なら「距離 XXm」表示。
4. **スキャン**（📷）でバーコードからアイテム獲得（無料 5 回／日 + プラチナ結晶で追加）。
5. **ダンジョン突入** → HP は全回復、入場前のインベントリと装備をスナップショット。
6. **各フロアを攻略**、最終フロアでボス撃破 → 攻略成功でゴールド・経験値・ドロップを持ち帰り。
7. **敗北時** はダンジョン中に拾った装備・素材・回復薬は失われ、入場前の状態にロールバック。ゴールドは持ち越し。
8. **攻略済ダンジョン** は範囲内なら何度でも再戦可能（入場毎に部屋レイアウト・敵配置が runSalt で変わる）。
9. **オンライン対戦** ボタンから 1v1 PvP もしくは 2 人協力ボス討伐（Firestore で同期）。

## 対応バーコード形式と挙動

`@zxing/browser` を `EAN_13 / EAN_8 / UPC_A / UPC_E / CODE_128 / ITF` の数字系に絞って起動。アプリ側でさらに **「数字 8〜20 桁」** に限定（QR コード・英数 Code39 等は無視）。

| 形式 | 桁数 | 想定用途 | カテゴリ | アプリでの扱い |
|-----|------|---------|---------|------------|
| **JAN-13 / EAN-13** | 13 桁 | 日本・世界の商品バーコード | `product` | 通常レア度のアイテム |
| **JAN-8 / EAN-8** | 8 桁 | 小型商品の短縮版 | `product` | 通常レア度のアイテム |
| **UPC-A** | 12 桁 | 北米の商品 | `product` | 通常レア度のアイテム |
| **UPC-E** | 6〜8 桁 | UPC-A の短縮版 | `product` | 通常レア度のアイテム |
| **Code 128（数字）** | 可変 | レシート・配送伝票・チケット | `receipt` | **レア度 +1 段階の底上げ** |
| **ITF** | 偶数桁 | 書籍流通・段ボール | `receipt` | **レア度 +1 段階の底上げ** |

> JAN コードは EAN-13 と完全互換（日本コードは先頭が `45` または `49`）。

### スキャン回数制限（無料 5 回／日 + 課金通貨）

`scan-budget.js`：

- 1 アカウントあたり **1 日 5 回まで無料**でスキャン可能
- 上限超過時は **プラチナ結晶 1 個**を消費して +1 回
- プラチナ結晶は課金専用通貨（ダンジョン報酬では入手不可。デバッグパネル or 「プラチナ結晶を購入（テスト）」ボタンで `PLATINUM_STUB_GRANT` 個付与）
- 日次リセットはローカル時刻の日付境界（YYYY-MM-DD）。状態は Firestore セーブで別端末にも引き継がれる

## バーコード → アイテム生成ルール

スキャンで得られるアイテムは **完全に決定論的**（同じバーコードなら同じアイテム）。

### バーコード桁の役割（13 桁正規化後）

8〜12 桁のコードは内部で **先頭ゼロ埋めで 13 桁化** されてから読まれます。

| 桁位置 | 影響先 | ロジック |
|-------|--------|---------|
| **4〜5 桁目** | アイテム属性 | `% 6` で属性選択 |
| **6 桁目（薬・3 種別）** | HP 薬 / MP 薬 / 状態異常回復薬 | `% 3` |
| **7 桁目（巻物カテゴリ）** | 鍵 / 薬 振り分け | `% 4 === 0` で鍵 |
| **8 桁目** | バーコードダンジョン入口抽選 | `% 8 === 0`（12.5%）で `dungeonPortal` |
| **全桁合計** | アイテム種別 | `% 4` で 武器/防具/回復薬/巻物 |
| **末尾 1 桁** | レアリティ | 表参照（レシート系は +1 段階） |

### レアリティ（末尾 1 桁）

| 末尾桁 | レアリティ | ステータス倍率 | スキル付与 |
|-------|----------|------------|---------|
| 0〜4 | コモン (灰) | ×1.0 | なし |
| 5〜7 | レア (青) | ×1.7 | 1 種類（クリ率/反射 等） |
| 8 | エピック (紫) | ×2.6 | 1 種類（高威力スキル） |
| 9 | レジェンド (金) | ×4.2 | 1 種類（強力固有スキル） |

> Code 128 / ITF（レシート系）は上記の **末尾桁から +1 段階上**のレア度に底上げ。

### アイテム種別（全桁合計 % 4）

| 値 | 種別 | 内容 |
|---|------|------|
| 0 | 武器 | 短剣／剣／斧／槍／魔法杖。属性つき・レア度で名前接頭辞・接尾辞・付属スキル変化 |
| 1 | 防具 | 盾／鎧／マント。属性つき・レア度で名前接頭辞・接尾辞・付属スキル変化 |
| 2 | 回復薬 | 6 桁目で HP / MP / 状態異常回復に分岐 |
| 3 | 巻物枠 | 7 桁目が 0 なら鍵、それ以外は薬に振り替え（巻物そのもののドロップは現状無効化中） |

### 装備の付属スキル例

```
コモン:    火の剣              ATK +5 （スキルなし）
レア:      火の剣・改          ATK +8 / 鋭刃: クリティカル率+15%
エピック:  火の剣・烈          ATK +12 / 烈火連斬: 攻撃時+7火ダメージ
レジェンド:火の剣・神威        ATK +20 / 龍王の咆哮: 攻撃時+15火ダメージ
```

> 名前の接頭辞・接尾辞・付属スキル名はレア度別の候補から決定論的に選択されます。

### バーコードダンジョン入口（`dungeonPortal`）

スキャン時に **`digits[8] % 8 === 0`（約 12.5%）** で当たる特殊アイテム。持ち物に格納され、メニューから「突入する」で **バーコード固有の一回限りダンジョン**へ。テーマ・属性・レアリティ・階数（2〜4F）はバーコード桁から決定論的。同じ商品を再スキャンすれば再挑戦可能。

### 伝説の書（`legendaryTome`）

ミニオンの試練ダンジョン入口アイテム。読むと該当ミニオン専用の特殊ダンジョン（3F・エピック級）が生成され、最上階の「◯◯王」を倒すと **そのミニオンが仲間入り**（実装は `_maybeRecruitMinion` / `buildSpecialDungeonForTome` / `generateMinionBoss`）。

## ダンジョン生成（位置ベース固定湧き）

地図は **約 200m × 200m のグリッド** に区切られ、各セルは `hashString('grid:<gx>:<gy>')` をシードにして決定論的にダンジョンが配置されます。

| 要素 | ロジック |
|------|---------|
| 出現可否 | 各セル 60% でダンジョン出現 |
| 配置位置 | セル内のランダム位置（中央寄り 30〜70%） |
| テーマ | 5 種類（古代遺跡／氷の洞窟／溶岩洞／魔法図書館／毒の沼地） |
| 属性 | 6 属性からランダム（火／水／草／雷／光／闇） |
| レアリティ基準 | コモン 45% / レア 35% / エピック 15% / レジェンド 5% |
| フロア数 | コモン 1〜3 / レア 2〜5 / エピック 4〜7 / レジェンド 6〜10 |
| 敵職業 | バーコード 3 桁目 `% 8` で 8 職業から決定論的に選択（同じダンジョン内は同職業） |
| 入場可能距離 | プレイヤー位置から **80m 以内** |
| 内部敵レベル | レアリティで決定（コモン Lv1 / レア Lv8 / エピック Lv25 / レジェンド Lv50）。**階層が深くても敵は強くならない**（行きたい強さのダンジョンを選ぶ運用） |
| ボス | 基準レアリティから **+1 段階** 昇格 + Lv +5。命名は属性プレフィックス + 「王」 |
| 部屋／配置 | 入場ごとに `runSalt` を混ぜて部屋レイアウト・敵位置・床アイテム位置がシャッフル |

### ダンジョン内に出るもの

- **モンスター**: 部屋ごとに 1〜2 体配置。最終フロア最後の部屋がボス
- **床アイテム**: 薬（通常落ち）／鍵／技の書（10%）／宝箱（18%・武器 75% / 防具 25%）／ゴールドの山（28%）
- **商人 NPC**（30% 確率）: 最終フロア以外に出現。タップで購入モーダル、攻撃すれば Lv +30 の超強敵として撃破 → 在庫全奪取

### 道端エンカウント（マップ上）

`MAP_ENCOUNTER_GRID = 0.001`（約 100m 格子）。プレイヤーから **80m 以内**に来た瞬間にピンが現れる（ヒステリシス +20m で抜ける）。

| 種別 | 出現重み | 内容 |
|------|---------|------|
| 👾 モンスター | 30% | 1 ルーム戦闘ステージ。プレイヤー Lv ±2 |
| 🎁 宝箱 | 8% | 武器 75% / 防具 25%。拾うと持ち物に入り鍵で開封 |
| 🧝 商人 | 4% | 在庫 4 点。攻撃も可（Lv +25） |
| 👹 強敵 | 3% | プレイヤー Lv +5〜+10。挑戦は任意 |
| —（空セル） | 55% | 何も出ない |

消費済み（拾った宝箱・倒した強敵）の seed セットは `localStorage` に保持されてピンが再出現しない。商人／通常モンスターは消費しない（同じ場所に戻ればまた遭遇）。

## モンスター職業（8 種）

`monster-jobs.js`：バーコード 3 桁目から決定論的に選ばれる。各職業は固有のステータス補正・好む範囲・固有 AI ヒント・命中時の状態異常付与を持つ。

| 職業 | 絵文字 | HP×ATK×DEF | AI ヒント | 命中時付与 |
|-----|------|---------|---------|---------|
| 獣王（ビースト） | 🦁 | 1.30 / 1.20 / 1.00 | rush（近接突進） | fracture 10% |
| 武道家（モンク） | 🥋 | 1.00 / 1.15 / 0.95 | doublehit（3T 毎の 2 連撃） | fracture 15% |
| ホラーマン | 👻 | 0.90 / 1.10 / 0.80 | phaseseal（封印付与） | confuse 18% |
| コウモリ（バット） | 🦇 | 0.75 / 0.90 / 0.75 | line3（直線 3 マス飛び道具） | spasm 20% |
| スケルトン | 💀 | 1.05 / 1.00 / 1.20 | rush | fracture 12% |
| 蛇族（サーペント） | 🐍 | 0.95 / 1.10 / 0.90 | pierce（直線貫通牙） | poison 25% |
| ゾンビ | 🧟 | 1.40 / 0.95 / 0.90 | regen（毎ターン 4% 回復） | sleep 8% |
| ドラゴン | 🐉 | 1.50 / 1.30 / 1.20 | breath（5T 毎の正面 5 マスブレス） | burn 28% |

属性プレフィックス（フレイム／アクア／ポイズン／サンダー／シャイン／シャドウ）と組み合わせて動的命名：「フレイムビースト」「シャドウスケルトン」など。

### 敵 AI の挙動

- **ターゲット選定**: ミニオンとプレイヤーの最寄り（マンハッタン距離）を狙う
- **遠距離技**: 視線（LOS）が通っていればチャージ後に LINE3 / LINE5 / PIERCE / DIAG 飛び道具
- **チャージ**: chargeBonus + 標準 3T で発動
- **状態異常付与**: 命中時に確率ロール。遠距離技は確率 1.3 倍

PP は MP コストに換算（PP30→MP4、PP1→MP60）。Lv77 メテオなどの究極技は MP 60 と高コスト。

### 技スロット

最大 **4 スロット**。レベル要件（コモン 1 / レア 5 / エピック 15 / レジェンド 30、ウィザード技は `learnedAt`）を満たせばセット可能。タイプ変更時はスロットから適性外を自動除外。

## オンラインプレイ（PvP / 協力）

`multiplayer.js` + Firestore コレクション `pvpRooms/{6 桁コード}`。

### モード

- **⚔️ 対戦（PvP）**: 1v1 ターン制 21×19 アリーナ。「現在の装備」または「セット装備（ATK +25 / DEF +15 で公平化）」を選択可能
- **🤝 協力（Co-op）**: 6 種のボス NPC を 2 人で討伐。ボスは固定スペック（火竜／水妖／雷神／大地の巨人／影の王／天使）

### フロー

1. ホストが 6 桁コード生成（衝突時は再試行）／ ゲストがコード入力で参加
2. ロビーで両者「準備 OK」 → ホストが `startBattle` で battle 遷移
3. ターン制で `submitMove` / `submitArenaAttack` / `submitOwnState` / `submitBossUpdate` を書き込み、相手は `onSnapshot` で同期
4. 勝敗確定で `state='finished'`。OK 後に `resetForRematch` で再戦も可能
5. ハートビート（`lastSeen`）と離脱検知（`handleHostLeave` / `handleGuestLeave`）で接続落ち対策

> ジャッジはクライアント信頼（HP 負数化で即勝利等の不正は防げない）。Phase 2 で Cloud Functions に判定を移す予定。

## セーブと認証

`save.js` / Firebase Auth + Firestore：

- メール／パスワード または Google アカウントでログイン
- ログイン状態は `browserLocalPersistence` でブラウザに永続化
- セーブは `saves/{uid}` に書き込み（30 秒毎にオートセーブ、`autoSave` 関数）
- 別端末でもログインすれば続きから（スキャン回数も引き継ぎ）

Firestore Security Rules（`firestore.rules`）：

- `saves/{uid}` は本人のみ読み書き可
- `pvpRooms/{code}` は認証済みなら誰でも読める。書き込みはホスト本人 / ゲスト本人 / 新規参加（guest=null → 自分）

## 開発フロー

設計書（`documents/` ディレクトリ）は `.gitignore` 済み（リポジトリに含めない）。次セッション再開用ログは `PROGRESS.md` を参照。

### ブランチ運用

`feat/xxx` / `fix/xxx` ブランチ → コミット → main マージ → push → `firebase deploy --only hosting` の順。`Bash(firebase deploy*)` は `.claude/settings.json` で許可済み。

### 新機能を追加するときは「デバッグモード対応」を必ず考慮する

このアプリは GPS とカメラに依存するため、**デバッグモード（`?debug=1`）から動作確認できない機能は実機テスト無しで検証不能** になります。新機能を追加する際は、以下を必ず一緒に実装してください：

| 機能の種類 | デバッグ側で必要な対応 |
|-----------|---------------------|
| **GPS に関わるもの**（移動・距離・スポーン等） | モック GPS で動作することを確認。必要なら `setPlayerPosition` で位置を動かせるように |
| **カメラ・スキャンに関わるもの** | モックスキャンの入力（テキスト＋フォーマット）で同じフローが流れるように |
| **永続化・ストレージに関わるもの** | デバッグ用キーは本番と分離（例: `debug:` プレフィックス）。リセットボタンをデバッグパネルに |
| **時間・期間に関わるもの** | 時刻を倍速にするか即時化するトグルをデバッグパネルへ |
| **戦闘・モンスター関連** | 戦闘テスト用の即時遭遇ボタン、HP/ATK 上書き、敵生成パラメータ強制指定など |
| **画面遷移・モーダル** | 各画面に直接ジャンプできるショートカットをデバッグパネルへ |

**実装パターン**:

1. 機能のコアロジックを純粋関数化（環境依存しない）
2. 環境依存部分（GPS / カメラ / storage）は最小に閉じ込める
3. デバッグパネルから上記の差し替え点を直接叩けるようにする
4. README のデバッグモード節に新規操作を追記

> 「実機にスマホを持って外を歩かないと検証できない」状況を作らないこと。
